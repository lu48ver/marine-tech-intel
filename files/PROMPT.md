# Claude Code 主提示詞 (PROMPT.md)

> 把下面整段複製,貼到 Claude Code 對話框,按 Enter。Claude Code 會讀取 `spec.md`、規劃步驟、然後動手做。

---

## 貼這段給 Claude Code 👇

```
請幫我建立一個海事技術情報彙總網站,完整規格請先讀 ./spec.md。

執行原則:
1. 先讀 spec.md,完整理解需求後,提出實作計畫並等我確認
2. 確認後再開始寫程式,不要一次寫完所有檔案,每完成一個模組就停下來讓我檢查
3. 全程用繁體中文回應,程式碼註解用英文
4. 遇到不確定的設計決策,問我而不是猜測
5. 每個爬蟲完成後實際跑一次驗證,確認能抓到資料才繼續下一個
6. 最後產出一份 README.md,寫清楚怎麼本機跑、怎麼部署、怎麼新增來源

我的環境:
- macOS / Windows (你判斷,問我也可以)
- 已安裝 Python 3.11+, Node.js, git
- GitHub 帳號已準備好,要部署到 GitHub Pages
- 我是工務監督實務人員,Python 我懂基礎,前端我會看不會寫

先做這幾件事:
1. 確認你讀過 spec.md
2. 提出你打算建立的資料夾結構
3. 列出第一階段 (MVP) 你打算先做哪 3 個爬蟲
4. 等我點頭再動手
```

---

## 為什麼這樣寫? (給你看的、不用貼給 Claude Code)

這份提示詞用了五個有效的技巧:

**1. 「先讀 spec.md」**
把詳細需求寫在獨立檔案,提示詞只是觸發。好處:規格可以慢慢改,不必每次都打一大段。

**2. 「提出計畫等我確認」**
這一句最重要。Claude Code 預設會直接動手,加上這句之後,它會先展示思路、列檔案清單,你看了不對可以馬上修正方向,不會等寫完才發現走偏。

**3. 「每完成一個模組停下來」**
避免它一次生 20 個檔案、你看不完。一步一步來,每步可以驗證。

**4. 「遇到不確定就問」**
Claude Code 預設是「猜並繼續」。明確要它問,可以避免後續返工。

**5. 「實際跑一次驗證」**
要求它跑程式而不只是寫程式。爬蟲常常網站結構跟想像不同,跑一次才知道。

## 後續會用到的指令範本

完成 MVP 之後,你會需要追加指令。這裡先給你幾個常用的:

### 新增來源
```
請新增一個來源:[網站名稱],URL: [...]
更新方式:[爬整頁/RSS/PDF 列表]
更新頻率:[每日/每週/每月]
請依照 spec.md 的爬蟲模式新增,並執行一次測試。
```

### 修 bug
```
[來源名稱] 的爬蟲今天跑出來是空的,請幫我:
1. 實際打開該網站看結構是否變動
2. 比對你的爬蟲程式碼
3. 修正後跑一次驗證
4. 把修正內容寫進 CHANGELOG.md
```

### 上 GitHub Pages
```
請幫我設定 GitHub Actions:
1. 每天 UTC 00:00 跑爬蟲
2. 把產出的 JSON 與 HTML commit 到 gh-pages 分支
3. 確認 GitHub Pages 設定指向 gh-pages
4. 在 README 寫清楚我要去 repo settings 哪裡點哪幾個按鈕
```

### 加新議題分類
```
我要新增一個議題分類「[議題名稱]」,優先級 HIGH。
請:
1. 在 topics.json 加入該議題的描述、優先級、相關來源
2. 在前端 hot topics 區塊顯示
3. 把所有來源的爬蟲輸出加上 tagging,讓相關文章自動歸類到此議題
```
